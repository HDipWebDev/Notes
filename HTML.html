Inside <body> in HTML5

Once inside the <body> section, HTML5 provides a number of tags that allow us to construct the the web page in a consistent manner. These are:


<header>...</header>
<nav>...</nav>
<article>...</article>
<aside>...</aside>
<section>...</section>
<footer>...</footer>

This is a considerable improvement over HTML4, where each of these items would typically been enclosed with a <div>, or division.

header / nav / article / aside / section / footer

  • <header> Contains the site headers, often banner images and the like
  • <nav> Contains navigation elements such as a menu
  • <article> Contains a self contained composition, such as news article.
  • <aside> Contains information that can be left out or ignored without affecting the composition of the article. This would typically be advertising.
  • <section> Contains a generic section of a composition or document.
  • <footer> Contains the site footer. Typically contains contact or copyright information.

Other Major HTML Elements

  • <p>...</p> Paragraph
  • <ol>...</ol> Ordered List (typically, numbered)
  • <ul>...</ul> Unordered List
  • <li>...</li> List Item
  • <table>...</table> Table
  • <tr>...</tr> Table Row
  • <li>...</th> Table Headings (column headings)
  • <td>...</td> Table data (or cell)
  • <a>...</a> Anchor

Headings <h1> to <h6>

  • <h1>...</h1> Headings must be enclosed in opening and closing tags
  • <h1> to <h6> Defines a hierarchy of importance. It is not intended as styling
  • When used in conjunction with the HTML5 <section> tag, difficulties may arise.
  • The <h1> tag is normally indexed by robots, and therefore should contain the most important information on your page. It is generally a bad idea to use images for headlines.
  • The <h1> to <h6> tags provide additional structure to page for robots, making them easier for robots to index.

Tables

Tables are constructed using a combination of tags organized in a specific manner.

  • <table>...</table> encompass all of the elements that define a particular table
  • <tr>...</tr> Is then used to define a table row
  • <th>...</th> Is used to define each column heading.
  • <td>...</td> Is used to define column data.

<div> Division

Used to create divisions and/or sections

  • <div>, when used in conjunction with CSS, forms the basis of website layout.
  • each <div> is placed in a structure that allows for the logical division of the webpage.
  • every <div>; should be given a unique id.
  • every <div> should be closed with a </div>.
...

We are going to spend a lot more time on <div> when dealing with CSS

<span>

Used to create in-line sections

  • <span> is often used to style elements contained within a <div>
  • <span> is used extensivly with CSS.
  • every <span> should be closed with a </span>.
  • like <div>, it could be considered as a structural element

<img> Image

Used to insert raster (bitmap) images into HTML. It does not have a closing tag. 

<img src='moon.jpg' alt='NASA image of footprint on the moon'>

<canvas>

  • Used to draw directly into the browser.
  • It should always be used with an id
  • The image is constructed in a script and sent to the canvas.
  • canvas does not maintain individual id for its elements, so is not as flexible as svg.
  • it is faster than svg.

Some really cool experiments are available at http://net.tutsplus.com/articles/web-roundups/21-ridiculously-impressive-html5-canvas-experiments/

<canvas>

<svg> Scalable Vector Graphics

  • Like canvas, it is used to draw directly into the browser.
  • Unlike canvas, each element that makes up the image has its own id
  • The image is constructed in a script and drawn to the svg container.
  • As each element is part of the DOM, it can be slow to render.
  • Image will not degrade when scaled.
  • quite easy to set-up for responsive design

<video>

Standard way to put video into HTML documents

Used in conjunction with the <source> tag, which will specify details of the media asset.


<video width='640' height='480' controls>
	<source src='edhd512.mp4' type='video/mp4'>
	<source src='edhd.ogv' type='video/ogg'>
	Your browser does not support HTML5 video.
</video>
