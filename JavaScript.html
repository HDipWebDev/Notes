JavaScript

  • JavaScript is an object orientated scripting language.
  • All major browsers have a JavaScript interpreter built-in.
  • Because it is built-in, we generally have cross-browser support. However, not all browsers implement everything in the same way.

JavaScript Variables

  • JavaScript is a loosely typed language
  • This means that when you declare a variable you do not have to specify its data type
  • Variable scope can be problematic if you do not exercise care.

JavaScript

  • JavaScript can be Server-Side or Client Side
  • We will stick to the far more common Client Side.
  • JavaScript is not Java!

JavaScript Output

JavaScript has three main outputs

  1. On your HTML page
  2. Popup boxes; Alerts etc
  3. Browser console.

JavaScript Popups

Popups are generally an annoyance to users, and should be used with caution. There are three types available

  1. Alert Box - Displays Message
  2. Confirm Box - OK, Cancel
  3. Prompt Box - Entered Text

JavaScript Alert Box

The Alert box simply displays a message. No further work is required. The example below will stop the page load until the user confirms.


<head>...
  <script>
    function fireAlert()
      {
        alert("I am an alert box!");
      }
    window.onload=fireAlert();
  </script>...
</head>

This is often referred to as 'blocking code' as it will block execution of everything else until it's done

JavaScript Confirm Box

A confirm box allows us to get a binary response; either OK or Cancel. We can then use this information to do other things, such as log the user selection to the console 


var response = confirm('Press OK to continue');
if (response == true){
  console.log('User Responded with OK');
} else {
  console.log('User Responded with Cancel');
}

JavaScript Prompt Box

A Prompt Box allows for the return of a string object from the box. Default values can be set for the response. 


var response = prompt('Please enter your name', 'Paul Vesey');
if (response != null && response != ''){
  console.log(response);
}

Comparison Operators Value and Type

As JavaScript is a loosely typed language, we can run into problems with variable types.

  • We can for instance assign a value of 32 to a variable by entering var x = 32;
  • All input from a Prompt box is in the form of a string, so if we had say var num = prompt('Enter a number');, adding x to num would yield unpredictable results.
  • For this reason the === test is important

Event Listeners and Handlers

  • Event Listeners are typically synchronous functions or methods that detect events or actions. Once an event is detected, the listener normally calls a Handler
  • Event Handlers are functions or methods that are called in response to a listener. This is where the main work is performed
  • Separating Listeners from Handlers allows for greater flexibility and makes maintenance easier

JavaScript Listeners and Handlers

A number of approaches have been developed for adding listeners and handlers for web. Most are now considered poor practice and should be avoided; they are included here as they do persist, in spite of the fact that newer approaches are easier to write and maintain. For the most part they can be considered as:

  1. In-line with anonymous JavaScript function
  2. In-line with named JavaScript function (internal or external JS)
  3. DOM Level 2, (no in-line JavaScript)

Inline Implementation

DO NOT USE: Whilst this approach will work, it is extremely difficult to debug and maintain. It is normally achieved by adding a listener directly to the HTML element. Whilst this approach is easy to read and understand, maintenance and debugging is extremely difficult on all but the most simple of websites.


<div class='grd12' id='FindMe' onclick='SayHello()'>

DOM Level 2 Implementation

The DOM Level 2 implementation is done entirely from JavaScript, with the exception of the element ID which is necessary for to identify the HTML element, in this case 'W3C'.


document.getElementById('W3C').addEventListener('click', cngColor, false);

DOM Level 2 Implementation Explained


document.getElementById('W3C').addEventListener('click', cngColor, false);
  • document.getElementById('W3C') selects the element that we want to add a listener to.
  • .addEventListener('click', cngColor, false); adds an event listener that will call the function chgColor when the user clicks the element.
  • the false option specifies that useCapture is not to be used. useCapture is a DOM Level 3 option that allows for greater control of how events are dispatched to handlers. You should specify either true or false.

And then... Internet Explorer

Older versions of IE do not support addEventListener. Instead they implement attachEvent. The JavaScript code is:


document.getElementById('W3C').attachEvent('onclick', cngColor);

You will also note that click is replaced by onclick, and that the event capture is not implemented.

And then... Internet Explorer

In order to overcome these inconsistencies, we need to create more robust mechanisms for attaching events to HTML elements. A typical approach is to develop an addEvent function in JavaScript that will handle both the W3C addEventListener, and the IE attachEvent. One approach for achieving this is available on Moodle

JavaScript Objects

JavaScript Objects can be created in a number of ways.


var myObject = new Object();
    // new instance of Object

Instantiate a new Object

Object Literal

Object is defined and created at runtime.


var car = {
	color: "blue",
	doors: 4,
	velocity: 60,
	faster: function(){
		return this.velocity + 10;
	}	
};

Constructor Function

Create an Object template.


function car(color, doors, velocity){
	this.color = color;
	this.doors = doors;
	this.velocity = velocity;
	this.faster = function(){
		return this.velocity += 10;
	};	
};

No Inheritance in JavaScript

JavaScript implements prototyping. Prototyping allows you to modify an object that has already been instantiated.


var myCar = new car('red', 2, 220);
console.log(myCar);
myCar.faster();
console.log(myCar);
car.prototype.stop = function(){
	return this.velocity = 0;
};
var myOtherCar = new car('blue', 4, 100);
myCar.stop();
myOtherCar.stop();
console.log(myCar);
console.log(myOtherCar);

.constructor Property

You can find the constructor function of an object by accessing its 'constructor' property. This can be useful when used in conjunction with prototyping


myCar.constructor.prototype.lightOn = function(){
		return this.lightsOn = true;
}

Adding properties and methods

JavaScript makes it easy to add properties and methods at runtime.


myCar.seats = 2;
myCar.brake = function(){
	return this.velocity -= 10;
}
console.log(myCar);

Prototyping v Adding Properties and Methods

Depends what you want to do:

If you want to add a property and/or method to all instantiated objects, and all future objects, then use prototyping.

If you want to effect one or a small number, then add properties and methods.

Variable Scope

JavaScript implements 2 scopes

  1. Local - when variable is declared within a function
  2. Global - everywhere else

Note that JavaScript does not implement Block level scope.

Variable Scope

Run the code below and note that block level scope is not implemented.


var a = 'Global-a';
for (var i = 0; i < 5; i++){
	var b = 'Global-b ';
	console.log(a, b, i);
};
console.log(a);	
console.log(b);
console.log(i);

Variable Scope

Run the code below and note that ommiting var will create a global variable from within a function.


function myLoop(count){
	b = 'Global from Function!';
	var c = 0;
	console.log(c);
	for (var i = count; i > 0; i--){
		console.log(c);
		c += i;
	}
	return c;
};
var a = 'Global-a';
var d = myLoop(5);
console.log(a);	
console.log(b);
console.log(d); // changed order as c will break script
console.log(c);

What is ES6?

ES6 is shorthand for ECMAScript 2015. ECMA => European Computer Manufacturer's Association

It is called ES6, because its predecessor was ES5.. (ES1 was published in June 1997)

What is ES6?

JavaScript, JScript and ActionScript are implementaitons of ECMAScript

ES6 is not fully supported by browsers at this time. Current status can be reviewed at http://kangax.github.io/compat-table/es6/

Using ES6 now

ES6 introduces some powerful features that can help us now. Furthermore, ES6 support is improving all of the time.

To use ES6 now, we can turn to babel.js to run cross-compilation to ES5
